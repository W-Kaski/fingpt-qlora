## 详细步骤说明

### 步骤 1: 代码准备

```python
# 1. 编写训练代码
# 2. 配置超参数
# 3. 提交到 GitHub
git add -A
git commit -m "Add training notebook"
git push
```

### 步骤 2: 诊断测试

```python
# 诊断配置
training_args = SFTConfig(
    max_steps=10,           # 只跑 10 步
    save_strategy="no",     # 不保存
    eval_strategy="no",     # 不评估
    logging_steps=1,        # 每步记录
)

# 保存日志到文件
log_path = "results/diagnostic_log.txt"
sys.stdout = Tee(sys.__stdout__, log_file)
```

### 步骤 3: 诊断分析

```python
# 1. 检查训练速度
time_per_step = total_time / 10
print(f"Time/step: {time_per_step:.2f}s")

# 2. 计算吞吐量
tokens_per_step = batch_size * max_seq_length
throughput = tokens_per_step / time_per_step
print(f"Throughput: {throughput:.0f} tokens/sec")

# 3. 与基准对比
unsloth_benchmark = 1500  # tokens/sec
ratio = throughput / unsloth_benchmark
print(f"vs Unsloth: {ratio:.1f}x")

# 4. 分析原因
# - 梯度检查点: +30% 开销
# - 评估开销: +32.5% 开销
# - 序列长度: O(n²)
# - 批次大小: GPU 利用率

# 5. 提出优化方案
# - 减少 eval_steps
# - 减少 gradient_accumulation_steps
# - 减少 max_seq_length
```

### 步骤 4: 决策

```python
# 根据诊断结果决定下一步

if time_per_step < 10:
    # 速度可接受，继续全量训练
    print("✅ Speed acceptable, proceed to full training")
    action = "continue"
    
elif time_per_step < 15:
    # 速度可优化，尝试优化配置
    print("⚠️  Speed can be optimized")
    print("Suggested optimizations:")
    print("  - Increase eval_steps: 100 -> 500")
    print("  - Decrease gradient_accumulation_steps: 8 -> 4")
    print("  - Decrease max_seq_length: 2048 -> 512")
    action = "optimize"
    
else:
    # 速度不可接受，重新设计
    print("❌ Speed unacceptable, need redesign")
    action = "redesign"

# 如果需要优化，返回步骤 2
if action in ["optimize", "redesign"]:
    print("\nReturning to step 2: Re-run diagnostic with new config")
    # 更新配置
    # 重新运行诊断测试
```

### 步骤 5: 全量训练


## 关键原则

### 1. 诊断循环是核心

```
诊断测试 → 诊断分析 → 决策 → 优化配置 → 诊断测试 → ...
    ↑                                              │
    └──────────────────────────────────────────────┘
```

**循环条件**：速度不可接受 且 还有优化空间
**退出条件**：速度可接受 或 已尝试所有优化

### 2. 不要创建多个 Notebook

```
❌ 错误做法:
- 03_diagnostic_test.ipynb
- 03_diagnostic_optimized.ipynb
- 03_diagnostic_v2.ipynb
- 03_diagnostic_v3.ipynb
- 03_training_optimized.ipynb

✅ 正确做法:
- 03_training_qlora.ipynb (使用 Kaggle 版本管理)
  - Version 1: 基线配置
  - Version 2: 优化配置
  - Version 3: 继续训练
```

### 3. 复用 Checkpoint

```
❌ 错误做法:
- 每次都从头训练

✅ 正确做法:
- 使用 resume_from_checkpoint=True
- 从最新 checkpoint 继续
```

### 4. 深入分析诊断结果

```
❌ 错误做法:
- 只看总结表格
- 不分析原因

✅ 正确做法:
- 计算实际吞吐量
- 与基准对比
- 分析根本原因
- 提出优化方案
```

### 5. 保存日志到文件

```
❌ 错误做法:
- 依赖 Kaggle CLI 日志 (有延迟)

✅ 正确做法:
- 保存到 results/training_log.txt
- 实时查看: tail -f results/training_log.txt
```
