## 详细步骤说明

### 步骤 1: 代码准备

```python
# 1. 编写训练代码
# 2. 配置超参数
# 3. 提交到 GitHub
git add -A
git commit -m "Add training notebook"
git push
```

### 步骤 2: 诊断测试

```python
# 诊断配置
training_args = SFTConfig(
    max_steps=10,           # 只跑 10 步
    save_strategy="no",     # 不保存
    eval_strategy="no",     # 不评估
    logging_steps=1,        # 每步记录
)

# 保存日志到文件
log_path = "results/diagnostic_log.txt"
sys.stdout = Tee(sys.__stdout__, log_file)
```

### 步骤 3: 诊断分析

```python
# 检查指标
1. 训练速度: s/step
2. Loss 趋势: 是否下降
3. VRAM 使用: 是否超限
4. 吞吐量: tokens/sec

# 与基准对比
Unsloth 基准: 1,500-3,500 tokens/sec
我们的结果: X tokens/sec
差异: Y 倍

# 分析原因
- 梯度检查点: +30% 开销
- 评估开销: +32.5% 开销
- 序列长度: O(n²)
- 批次大小: GPU 利用率
```

### 步骤 4: 决策

```python
# 根据诊断结果决定下一步

if speed < 10s/step:
    # 速度可接受，继续全量训练
    action = "continue"
elif speed < 15s/step:
    # 速度可优化，尝试优化配置
    action = "optimize"
else:
    # 速度不可接受，重新设计
    action = "redesign"
```

### 步骤 5: 全量训练

```python
# 使用 Kaggle 版本管理
# 不要创建新 notebook，使用同一个 notebook 的不同版本

# 如果有 checkpoint，使用 resume
checkpoints = sorted(glob.glob("outputs/*/checkpoint-*"))
if checkpoints:
    trainer.train(resume_from_checkpoint=checkpoints[-1])
else:
    trainer.train()
```

### 步骤 6: 训练监控

```python

# 可以在另一个终端运行:
tail -f results/training_log.txt

- 梯度范数
- checkpoint 保存
```


## 关键原则

### 1. 不要创建多个 Notebook
```
❌ 错误做法:
- 03_diagnostic_test.ipynb
- 03_diagnostic_optimized.ipynb
- 03_diagnostic_v2.ipynb
- 03_diagnostic_v3.ipynb
- 03_training_optimized.ipynb

✅ 正确做法:
- 03_training_qlora.ipynb (使用 Kaggle 版本管理)
  - Version 1: 基线配置
  - Version 2: 优化配置
  - Version 3: 继续训练
```

### 2. 复用 Checkpoint
```
❌ 错误做法:
- 每次都从头训练

✅ 正确做法:
- 使用 resume_from_checkpoint=True
- 从最新 checkpoint 继续
```

### 3. 深入分析诊断结果
```
❌ 错误做法:
- 只看总结表格
- 不分析原因

✅ 正确做法:
- 计算实际吞吐量
- 与基准对比
- 分析根本原因
- 提出优化方案
```

### 4. 保存日志到文件
```
❌ 错误做法:
- 依赖 Kaggle CLI 日志 (有延迟)

✅ 正确做法:
- 保存到 results/training_log.txt
- 实时查看: tail -f results/training_log.txt
```
